## 1. 定价策略分析

### 1.1 四种模式对比

| 模式 | 描述 | 优势 | 劣势 | 适用阶段 |
|------|------|------|------|---------|
| **完全免费 (Free)** | 开源插件，用户自带 API Key | 传播快，安装无门槛 | 无直接收入，难以持续 | 冷启动期 |
| **Freemium** | 基础功能免费 + 高级功能付费 | 低门槛获客 + 高级用户转化收入 | 需要明确的 free/paid 功能边界 | 成长期 |
| **订阅制 (Subscription)** | 按月/年订阅 | 稳定可预测收入 | 门槛高，初期转化难 | 成熟期 |
| **按量付费 (Pay-as-you-go)** | 按 Token 消耗计费 (插件加价代理) | 用多少付多少，公平 | 需要自建 API 代理层，运维成本高 | 规模化后 |

### 1.2 竞品定价对标

| 产品 | 定价模式 | 价格 | 包含内容 |
|------|---------|------|---------|
| **GitHub Copilot Individual** | 订阅 | $10/月 或 $100/年 | 代码补全 + Chat + 基础 Agent |
| **GitHub Copilot Business** | 订阅 | $19/月/人 | 企业管理 + 安全合规 + IP 保障 |
| **Cursor Pro** | 订阅 | $20/月 | 500 次快速请求 + 无限慢速请求 |
| **Cursor Business** | 订阅 | $40/月/人 | 团队管理 + 合规 + 优先支持 |
| **Cline (Claude Dev)** | 免费 + 自带 Key | $0 | 开源，用户自付 API 费用 |
| **Windsurf Pro** | 订阅 | $15/月 | AI Flow + Multi-file Edit |
| **Amazon Q Developer Pro** | 订阅 | $19/月 | Agent + Security Scan |

**关键洞察**：
- 头部玩家定价区间在 **$10-40/月**
- 开源工具 (Cline) 通过 "自带 Key" 模式获得大量用户，但无直接收入
- 订阅制是主流，但按量付费 (token-based) 在高级用户中更受欢迎

## 2. 推荐定价方案

### 2.1 分阶段定价策略

#### Phase 1-3: 开源免费 + 自带 Key (BYOK 模式)

```
King Agents Community Edition (社区版)
├── 价格: $0 (永久免费)
├── 模式: 用户自带 Anthropic/OpenAI API Key
├── 功能: 全部核心功能开放
│   ├── 5 Agent 完整协作
│   ├── 像素风战场可视化
│   ├── KDA + 段位系统
│   └── 对局历史
├── 限制: 无
└── 目标: 快速获取用户，建立口碑
```

**为什么选 BYOK 而非 Freemium**：
1. **零运维成本**：不需要自建 API 代理，不承担 LLM 调用费用
2. **对标 Cline 成功路径**：Cline 通过免费 + BYOK 在 3 个月内达到 10 万+ 安装
3. **建立信任**：中国开发者对付费订阅持谨慎态度，BYOK 消除付费顾虑
4. **用户自感知成本**：经济面板让用户直接看到 API 费用，建立正确的成本预期

#### Phase 4+: 引入增值服务 (可选)

```
King Agents Pro (专业版) — 未来考虑
├── 价格: $12/月 或 $120/年
├── 额外功能:
│   ├── 云端对局历史同步
│   ├── 团队段位排行榜
│   ├── 自定义像素皮肤 (Agent 外观)
│   ├── 高级数据分析面板
│   ├── 优先 Bug 修复和功能请求
│   └── 商用授权 + SLA
├── 仍然保持: 用户自带 API Key
└── 目标: 从高级用户获取可持续收入
```

```
King Agents Team (团队版) — 远期
├── 价格: $25/月/人
├── 额外功能:
│   ├── 团队共享 Agent 配置/Prompt
│   ├── 统一 API Key 管理
│   ├── Token 消耗审计报表
│   ├── 管理员控制台
│   └── SSO + RBAC
└── 目标: 企业级收入
```

### 2.2 定价理由

| 决策 | 理由 |
|------|------|
| 社区版永久免费 | 差异化竞争——Cursor/Copilot 都收费，我们免费全功能 |
| BYOK 模式 | 零 LLM 成本，健康的成本结构 |
| Pro 定价 $12/月 | 低于 Cursor ($20) 和 Copilot Business ($19)，降低付费门槛 |
| Team 定价 $25/月 | 对标 Cursor Business ($40)，有竞争力 |
| 像素皮肤作为付费点 | 游戏化的核心变现手段 (参考 LoL 皮肤模式) |

## 3. 成本估算

### 3.1 LLM API 成本 (以 Anthropic Claude 为基准)

#### 模型定价 (截至 2026 Q1)

| 模型 | Input ($/M tokens) | Output ($/M tokens) | 备注 |
|------|-------------------|--------------------|----|
| Claude Opus 4.6 | $15.00 | $75.00 | Router/Coder/Guardian 使用 |
| Claude Sonnet 4.6 | $3.00 | $15.00 | Builder/Scout 使用 |

> 注：以上为 Anthropic 官方定价，实际价格可能因 prompt caching、batch API 等优惠有所降低。

#### 单次 "对局" Token 消耗估算

**场景：中等复杂度功能开发（如 "添加验证码登录"）**

| Agent | 模型 | Input Tokens | Output Tokens | 单次成本 |
|-------|------|-------------|--------------|---------|
| Router | Opus 4.6 | ~3,000 | ~1,500 | $0.157 |
| Scout | Sonnet 4.6 | ~8,000 | ~3,000 | $0.069 |
| Coder | Opus 4.6 | ~15,000 | ~5,000 | $0.600 |
| Guardian | Opus 4.6 | ~12,000 | ~3,000 | $0.405 |
| Builder | Sonnet 4.6 | ~2,000 | ~1,000 | $0.021 |
| **合计** | — | **~40,000** | **~13,500** | **$1.252** |

#### 不同任务类型的成本区间

| 任务类型 | 典型 Token 消耗 | 估算成本 | 说明 |
|---------|----------------|---------|------|
| 简单任务（问答/小改动） | ~8,000 | $0.15-0.30 | 仅 Router + Scout + Coder |
| 中等任务（功能开发） | ~40,000 | $1.00-1.50 | 全员参与 |
| 复杂任务（大功能/重构） | ~100,000 | $3.00-5.00 | 多轮迭代 + Gank |
| 超大任务（架构级） | ~200,000+ | $8.00-15.00 | 多次团战 |

#### 用户月度成本估算

| 用户类型 | 日均对局数 | 月度 Token | 月度费用 (BYOK) |
|---------|-----------|-----------|----------------|
| 轻度用户 | 3-5 局 | ~3M tokens | $15-30 |
| 中度用户 | 10-15 局 | ~10M tokens | $50-80 |
| 重度用户 | 20-30 局 | ~25M tokens | $120-200 |

> 对比：Cursor Pro $20/月（500 次请求上限），King Agents 中度用户约 $50-80/月但无上限且功能更强。

#### 经济模式成本

将全部 Agent 切换到 Sonnet 4.6 (economy 预设)：

| Agent | 模型 | Input Tokens | Output Tokens | 单次成本 |
|-------|------|-------------|--------------|---------|
| 全部 5 Agent | Sonnet 4.6 | ~40,000 | ~13,500 | **$0.322** |

经济模式下成本降低约 **75%**，月度费用约 $12-60。

## 4. 盈亏平衡分析

### 4.1 Phase 1-3: 社区版 (无直接收入)

**策略**：不追求盈利，追求用户增长和口碑。

成本：~$15/月 (几乎为零)
收入：$0
亏损：~$15/月 (可忽略)

**关键假设**：项目由创始团队/开源社区维护，不计人力成本。

### 4.2 Phase 4+: 引入 Pro 版后的盈亏分析

#### 收入模型

假设 Pro 转化率 = 3%（参考 VSCode 插件行业基准 2-5%）

| 总安装量 | 月活用户 (MAU 30%) | Pro 付费用户 (3%) | 月收入 (MRR) |
|---------|-------------------|------------------|-------------|
| 5,000 | 1,500 | 45 | $540 |
| 10,000 | 3,000 | 90 | $1,080 |
| 30,000 | 9,000 | 270 | $3,240 |
| 50,000 | 15,000 | 450 | $5,400 |
| 100,000 | 30,000 | 900 | $10,800 |

#### 盈亏平衡点

```
月度固定成本: ~$850 (含云服务、运维)
Pro 月费: $12/用户

盈亏平衡 = $850 / $12 = 71 个 Pro 用户
需要总安装量 = 71 / 0.03 / 0.30 ≈ 7,900 安装

结论：当总安装量达到约 8,000 时，
      按 3% Pro 转化率，月收入可覆盖运维成本。
```

#### 乐观场景 (对标 Cline 增长速度)

Cline 在 2025 年达到 100,000+ 安装。假设 King Agents 凭借差异化 (可视化 + 游戏化) 在发布 6 个月后达到 30,000 安装：

```
30,000 安装 × 30% MAU × 3% Pro = 270 Pro 用户
MRR = 270 × $12 = $3,240/月
年收入 = ~$38,880
年成本 = ~$10,200
年利润 = ~$28,680
```

#### 远期场景 (引入 Team 版)

```
假设: 50,000 安装, 其中 1% 转化为 Team 版 (平均 5 人团队)
Team 用户 = 50,000 × 0.30 × 0.01 × 5 = 750 人
Team MRR = 750 × $25 = $18,750/月

总 MRR = Pro $5,400 + Team $18,750 = $24,150/月
年收入 = ~$289,800
```

### 4.3 成本优化策略

| 策略 | 节省幅度 | 可行性 |
|------|---------|--------|
| **Prompt Caching** | 减少 30-50% input token 成本 | Anthropic 已支持 |
| **Batch API** | 减少 50% 成本（非实时场景） | 适用于 Guardian 的审查任务 |
| **智能路由** | 简单任务只派 Scout + Coder，跳过 Guardian/Builder | Router 可判断 |
| **上下文精准裁剪** | 减少 20-30% input tokens | Scout 只传递相关代码段 |
| **Sonnet 优先 + Opus 兜底** | 总成本降低 40-60% | 提供 "智能切换" 预设 |

## 6. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| Anthropic 大幅涨价 | 低 | 高 | 多 LLM 支持，用户可切换 OpenAI/本地模型 |
| 竞品推出类似多 Agent 方案 | 中 | 中 | 游戏化 + 可视化是护城河，难以快速复制 |
| Pro 转化率低于预期 | 中 | 中 | 社区版已有完整功能，Pro 价值需持续验证 |
| 用户觉得 Token 消耗太高 | 高 | 中 | 经济模式 (全 Sonnet)；智能路由降低消耗；透明的成本面板 |
| 开源维护者 Burnout | 中 | 高 | 早期引入 Pro 收入；建设贡献者社区 |
